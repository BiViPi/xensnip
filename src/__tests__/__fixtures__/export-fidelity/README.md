# Export Fidelity Baselines

This directory contains the golden-image baselines for the Workstream 2 export
fidelity matrix. **Do not edit files here manually.**

## Generation environment

| Field | Value |
|---|---|
| OS | Windows 11 |
| Node version | v24.14.0 |
| `canvas` package version | 3.2.3 |
| `pixelmatch` package version | 7.2.0 |
| `pngjs` package version | 7.0.0 |
| Generation date | 2026-05-11 |
| Generated by | `UPDATE_BASELINES=1 pnpm test src/__tests__/exportFidelity/` |

## Structure

```
export-fidelity/
  arrow/           v1-plain-f1.png  v1-plain-f2.png  v2-*  v3-*  v4-*
  rectangle/       (same pattern)
  text/            (same pattern)
  blur/            (same pattern)
  pixelate/        (same pattern)
  opaque-redact/   (same pattern)
  numbered/        (same pattern)
```

56 PNG files total (7 types × 4 variants × 2 formats).
JPEG-format rows (`-f2.png`) are stored as PNG after decoding the JPEG output,
so all baselines are readable by `pngjs.PNG.sync.read`.

## Rebaselining

To regenerate baselines (after intentional renderer changes):

```sh
UPDATE_BASELINES=1 pnpm test src/__tests__/exportFidelity/
```

Then visually review each changed baseline PNG in `__fixtures__/export-fidelity/`
before committing.

## Notes

- Baselines were generated under the **headless node-canvas** renderer, not a
  real browser. Rendering may differ slightly from the Tauri WebView. Layer B
  (manual live-parity checklist) is the gate for browser fidelity.
- `blur` type uses `ctx.filter` which is not fully implemented in node-canvas.
  If blur rows show `blocked` in CI, update the tolerance or mark as `blocked`.
